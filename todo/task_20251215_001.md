# 微信视频号下载项目 Python 重构方案分析

## 1. 项目概览

本项目旨在提供一个便捷的微信视频号视频下载工具，支持批量下载、自动解密、断点续传以及下载记录管理。原项目基于 Go 语言开发，利用中间人攻击（MITM）技术拦截微信客户端流量，提取视频地址和解密密钥，并提供 Web 控制台进行任务管理。

## 2. 原版技术架构分析 (Go)

通过代码审计，原版实现的核心机制如下：

### 2.1 核心组件
*   **网络代理**: 使用 `SunnyNet` 库建立 HTTP/HTTPS 代理，拦截微信网络请求。
*   **证书管理**: 自动生成和安装 CA 根证书，实现 HTTPS 流量的解密看到明文。
*   **Web Server**: 内置 HTTP 服务，提供 API 接口（`/api/...`）和 WebSocket 服务（实时进度推送）。
*   **前端注入**: 在拦截到视频号特定页面（HTML）时，注入 `main.js` 等脚本。这些脚本负责在浏览器端提取视频元数据和 Key，并通过 API 发送回本地服务器。

### 2.2 关键业务逻辑
*   **解密算法**: 视频文件经过 XOR 加密。
    *   **算法**: `ISAAC64` (伪随机数生成器)。
    *   **流程**: 使用请求中获取的 `Key` (seed) 初始化 ISAAC64，生成密钥流。
    *   **应用**: 将视频文件的前 N 个字节（通常是 128KB 或更少）与密钥流进行 XOR 运算以还原数据。后续数据通常未加密。
    *   **参考代码**: `pkg/util/isaac64.go` 和 `internal/handlers/batch.go` 中的 `downloadAndDecrypt` 方法。
*   **下载机制**: 支持并发下载、断点续传（Range Header）。下载过程中一边下载网络流，一边进行 XOR 解密并写入磁盘。
*   **数据存储**: 使用 CSV 文件记录下载历史，SQLite (`console.db`) 存储 Web 控制台状态。

## 3. Python 实现方案

考虑到 Python 生态的丰富性，我们采用以下技术栈进行重构：

### 3.1 技术选型
*   **代理核心 (MITM)**: **`mitmproxy`**
    *   **理由**: Python 领域最强大的 MITM 工具，支持脚本扩展（Addons），易于处理 HTTPS 拦截、流量修改和注入。
*   **Web 后端**: **`FastAPI`**
    *   **理由**: 高性能、异步支持（对下载任务友好）、自动生成 API 文档，适合构建本地控制台 API。
*   **下载引擎**: **`aiohttp` + `aiofiles`**
    *   **理由**: 全异步 IO，能够高效处理多任务并发下载，且易于实现流式解密处理。
*   **数据库**: **`SQLite` + `Tortoise-ORM` (或 SQLAlchemy)**
    *   **理由**: 轻量级本地存储，ORM 方便管理下载记录模型。
*   **UI 方案**: 沿用原版 `web/console.html` 或构建简单的 `Streamlit`/`NiceGUI` 界面。建议初期直接托管原版静态资源，后端兼容原版 API 接口。

### 3.3 关键模块详细设计

#### A. ISAAC64 算法移植
Go 版本的 `isaac64` 必须精确移植到 Python。
- **难点**: Go 的 `uint64` 溢出行为和位运算在 Python（大整数）中需要显式处理（使用 `& 0xFFFFFFFFFFFFFFFF` 模拟 64 位截断）。
- **目标**: 下一步任务的首要工作，确保生成的 XOR 密钥流与 Go 版本完全一致。

#### B. 代理与注入 (Mitmproxy Addon)
编写一个 `mitmproxy` 脚本：
1.  监听 HTTP 响应。
2.  匹配微信视频号域名/路径。
3.  在 HTML `<body>` 标签末尾注入 `<script src="http://127.0.0.1:port/inject/main.js"></script>`。
4.  拦截 `/api/` 开头的特定的上报流量，将其转发给 FastAPI，或者直接在 Addon 内部处理（推荐转发给 FastAPI 以解耦）。

#### C. 流式解密下载器
实现一个 `SmartDownloader` 类：
- 输入：URL, Key (Seed), Path.
- 逻辑：
    - 初始化 ISAAC64(Seed)。
    - 发起 `GET` 请求 (stream=True)。
    - 读取 Response Chunk。
    - 如果是文件头部（前 128KB），对 Chunk 进行 XOR 运算。
    - 写入文件。
    - 更新进度条状态。

## 6. 移动端实现方案与可行性分析

针对用户关心的“移动端落地”问题，我们需要面对 Android (7.0+) 和 iOS 对 SSL 证书信任机制的严格限制。

### 6.1 核心挑战：SSL Pinning 与 证书信任
*   **现状**: 现代 Android 系统（Android 7.0+）默认只信任系统级 CA 证书，用户安装的 CA 证书仅对浏览器有效，对大多数 App（包括微信）无效。
*   **难点**: 想要在手机 App（微信）内实现 HTTPS 抓包（MITM），通常需要 **Root 权限 (Android)** 或 **越狱 (iOS)** 来将 CA 证书移入系统信任区。否则，微信会提示网络错误或直接断开连接。

### 6.2 可行方案分析

#### 方案 A: 独立 App (Android VpnService) - **仅限 Root 用户**
*   **原理**: 开发一个 Android App，启动 VpnService 拦截流量，并在本地运行 Python (ChaquoPython/Termux) 核心逻辑。
*   **可行性**: ❌ **低 (针对普通用户)**。
    *   非 Root 手机无法解密微信流量。
*   **真实落地评分**: 20/100 (仅极客可用).

#### 方案 B: PC 代理辅助模式 - **高可行性**
*   **原理**: 手机连接 WiFi，设置 HTTP 代理指向 **PC (运行本 Python 程序)** 的 IP。
*   **流程**:
    1.  电脑运行 Python 代理服务。
    2.  手机安装电脑生成的证书（需在设置中信任）。
    3.  手机浏览视频号。
    4.  电脑端捕获 Key 并自动下载。
*   **可行性**: ✅ **高**。这是目前抓包工具（Charles/Fiddler）的标准用法。iOS 支持较好，Android 需要视微信版本而定（微信小程序/视频号有时会使用自己的网络栈，可能绕过代理设置，但通常 WebView 还是可以被抓到的）。
*   **真实落地评分**: 90/100.

#### 方案 C: 移动端 Web 版 + 浏览器插件 - **推荐 (免 Root)**
*   **原理**: 不在微信 App 内浏览，而是使用支持插件的手机浏览器（如 Android 的 **Kiwi Browser** 或 iOS 的 **Orion**），访问微信视频号 **Web 版** (`channels.weixin.qq.com`)。
*   **流程**:
    1.  在 Kiwi Browser 中安装 **Tampermonkey (油猴)** 插件。
    2.  安装适配的 UserScript (将 `main.js` 改写为油猴脚本)。
    3.  登录 web 版视频号，浏览视频。
    4.  UserScript 拦截数据 -> 发送给本地 Termux 服务 或 触发直接下载。
*   **可行性**: ✅ **高**。浏览器信任用户证书，且 JS 注入更直接，无需复杂的 MITM。
*   **真实落地评分**: 85/100.

#### 方案 D: 分享链接解析服务 - **服务器端实现**
*   **原理**: 用户在手机微信点击“分享 -> 复制链接”，粘贴到我们的小程序/网页。服务器端（Headless Browser）去访问该链接并提取。
*   **难点**: 微信链接通常有复杂的 Session 校验，服务器端难以模拟登录态，容易失效。
*   **真实落地评分**: 40/100.

### 6.3 结论
若要实现“真实落地”的移动端方案，最稳妥的路径依次是：
1.  **PC 代理模式**: 手机仅作为浏览端，核心逻辑仍在电脑，通过局域网代理连接。
2.  **移动浏览器 + 油猴脚本**: 完全脱离 PC，利用 Kiwi Browser 等工具的能力。

本项目 Python 重构优先支持 **PC 端 + 局域网代理** 模式，这也是最成熟的方案。
